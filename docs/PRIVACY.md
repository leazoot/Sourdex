# Privacy — Sourdex

> Sourdex 是本地优先（local-first）软件。本文件说明 Sourdex 如何处理你的数据。对齐 PRD §17 与 [.claude/rules/security.md](../.claude/rules/security.md)。

## 核心原则

- **默认本地存储。** 你保存的资料（正文、Markdown、原始快照、标签、索引）全部存放在你自己设备上的数据目录中。
- **默认不上传任何资料内容。** Sourdex 不会把你的资料发送到任何外部服务器。
- **无遥测、无账号。** v0.1 不收集使用数据、不需要登录、不联网回传。

## 数据存放在哪里

所有数据位于本地数据目录（可通过 `SOURDEX_DATA_DIR` 配置 / 迁移）：

- macOS：`~/Library/Application Support/Sourdex`
- Windows：`%APPDATA%\Sourdex`
- Linux：`~/.local/share/sourdex`

其中包含：SQLite 数据库（`sourdex.db`，含全文索引）、原始 HTML 快照与导出文件（`files/`）、访问 token（`config/auth.json`，权限 0600）。

## 本地服务与插件通信

- 本地服务**默认仅监听 `127.0.0.1`**（回环地址），不对局域网 / 公网开放。
- 浏览器插件访问本地服务需携带 token；**首次连接需用户通过配对码确认**（配对码仅显示在服务端控制台、5 分钟内有效、单次使用、不经网络返回）。
- CORS 仅允许浏览器插件与本地 Web UI 来源。

## AI 功能（v0.2，默认关闭）

- AI 摘要 / 标签 / 语义检索 / 问答为后续版本功能，**默认关闭**。
- 仅当你**主动配置** AI Provider 后，Sourdex 才会按你的操作把相关资料片段发送给你所配置的模型；发送内容会在设置中明确说明。
- 所有 AI 功能可随时关闭。不配置 AI 时，保存 / 阅读 / 全文搜索 / 导出均正常工作。
- API Key 不以明文写入普通配置文件、不写日志、不发送到前端长期保存（v0.2 优先使用系统 Keychain / 加密存储）。

## 日志

- 默认**不记录用户正文**；**不记录 API Key 或隐私字段**。
- 错误日志包含 request_id / job_id 以便排查；导出的诊断日志会脱敏。

## 删除与迁移

- 删除资料为**软删除**（标记状态），不会物理删除你的原始文件；删除前有二次确认。
- 数据目录可整体迁移到其他位置（文件路径相对数据目录存储）。

## 版权边界

- Sourdex 只保存你当前可访问的内容，不绕过付费墙、不提供公共资源分享库、不默认下载视频、不协助批量抓取受限内容。
- 导出内容保留来源 URL；你需遵守原网站的使用条款。

## 变更

本隐私说明随版本演进；涉及数据外发的任何变化都会在发布说明与本文件中明确标注。
