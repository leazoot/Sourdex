# Rules — Security & Privacy

> 适用范围：本地服务安全、鉴权、密钥、输入校验、隐私边界、版权边界。须符合 PRD 17 与 [docs/02_REQUIREMENTS.md](../../docs/02_REQUIREMENTS.md) §2.2。

## 隐私原则（PRD 17.1, 14.1）

- 默认本地存储，默认**不上传任何资料内容**。
- 仅当用户主动配置 AI Provider 后，才允许发送资料片段给模型；发送前必须在设置中明确说明发送内容。
- 所有 AI 功能可关闭；不配置 AI 时保存/阅读/全文搜索/导出必须正常工作。
- AI 输出不得覆盖原文。

## 本地服务安全（PRD 17.3）

- **默认监听 127.0.0.1，禁止默认监听 0.0.0.0。**
- 插件访问本地服务需 token；第一次连接需用户确认（机制见 OQ-A1）。
- CORS 只允许插件 ID 与本地 Web UI。
- **所有 API 输入必须用 Zod 校验。**

## API Key 存储（PRD 17.2，v0.2）

- 优先级：系统 Keychain > 本地加密文件 > 环境变量。
- 禁止：明文写普通配置文件；明文写日志；明文发送到前端长期保存。
- `provider_configs` 表不存明文 Key。

## 日志与脱敏（PRD 11.6, 17.1）

- 默认不记录用户正文；禁止记录 API Key 与隐私字段。
- 错误日志含 request_id/job_id 便于排查。
- 导出的诊断日志必须脱敏。

## 输入校验与错误暴露

- 扩展上送数据、API 请求、导入数据一律 Zod 校验。
- 不向客户端暴露底层错误堆栈；返回可读的用户级错误。

## 内容版权边界（PRD 17.4）

- 只保存用户当前可访问内容。
- 不做付费墙绕过；不提供公共资源分享库；不默认下载视频；不帮助批量抓取受限内容。
- 导出内容保留来源 URL；文档明确用户须遵守原网站条款。

## 数据安全

- 软删除不物理删用户文件；删除前二次确认。
- 数据目录路径可迁移；不静默删除用户资料。

## 禁止

- 默认监听 0.0.0.0 或允许任意来源 CORS。
- 明文存储/记录/外发 API Key。
- 在日志记录用户正文或隐私字段。
- 未校验外部输入。
- 默认上传资料内容给任何外部服务。
- 实现付费墙绕过或违规批量抓取。
