In the cloud, the data you create with an application is held on a server you don’t control. The promise of local-first software is to reverse that arrangement: your data lives on your own devices first, and the network becomes an enhancement rather than a dependency.

This is not a return to the era of files trapped on a single machine. Modern local-first apps keep working offline, sync seamlessly when a connection returns, and let several people collaborate on the same document in real time — all without a central authority owning the source of truth.

“The cloud cannot give us ownership and agency. For that, the data has to live on our own devices, under our own control.”

Seven ideals

The authors distill the movement into seven ideals: no spinners, work happens instantly; your work is not trapped on one device; the network is optional; collaboration is seamless; the long now — data outlives the company that made the app; security and privacy by default; and finally, you retain ultimate ownership and control.

Conflict-free replicated data types (CRDTs) are the technical foundation that makes much of this possible. They allow independent replicas to be merged automatically, without a server arbitrating every change, so two people editing offline can reconcile their work cleanly when they reconnect.

Sourdex note — This passage maps directly onto how Sourdex stores your sources: extracted, indexed, and searchable on disk, with the cloud kept strictly optional.

For a tool whose entire job is to remember things on your behalf, local-first is not a feature — it is the premise. An index you cannot lose, cannot be locked out of, and can export at any moment is the only kind worth building a memory on.

How do local-first apps resolve sync conflicts when editing offline?
{{ t.sources_ }}:
{{ t.answer }} {{ t.confidence }}: {{ t.high }} · {{ t.basedOn }}

Local-first applications avoid a central server arbitrating edits. Instead, each device keeps its own replica and records changes locally, so editing continues uninterrupted even while offline.1

When devices reconnect, conflict-free replicated data types (CRDTs) merge the divergent replicas automatically. Each operation carries enough causal metadata that merges are deterministic — every replica converges to the same result regardless of the order updates arrive in.12

Because the underlying store is often a single local file, syncing reduces to snapshotting and merging that state across devices, with no round-trip required before the user can read or search.3

Once the source is captured, search and citation become possible offline.
